# BobForge - AI App Factory

> Transform your ideas into complete application blueprints powered by IBM Bob

BobForge is an AI-powered application factory built for the IBM Bob Hackathon. It demonstrates IBM Bob as a comprehensive Software Development Life Cycle (SDLC) partner by generating complete application blueprints from simple project ideas.

## 🎯 Project Overview

BobForge takes a project idea and generates:
- 📋 **Product Requirements Document (PRD)** - Problem statement, target users, features
- 🏗️ **System Architecture** - Technology stack and component design
- 🗄️ **Database Schema** - Normalized tables with relationships
- 🔌 **API Specification** - RESTful endpoints with full documentation
- 🎨 **Frontend Plan** - UI pages and component structure
- 🧪 **Test Plan** - Comprehensive testing strategy
- 🚀 **Deployment Checklist** - Step-by-step deployment guide
- 🤖 **IBM Bob Build Prompt** - Ready-to-use prompt for building the app

## 🌟 Key Features

- **Instant Blueprint Generation** - Transform ideas into detailed specs in seconds
- **Complete SDLC Coverage** - From requirements to deployment
- **IBM Bob Integration** - Generate prompts optimized for IBM Bob
- **Export Capabilities** - Download blueprints as Markdown
- **Artifact Tracking** - Document Bob's contributions
- **Evidence Dashboard** - Showcase Bob as SDLC partner

## 🚀 Quick Start

### Prerequisites

- Node.js 16+ and npm
- Git

### Installation

1. **Clone the repository**
```bash
git clone <repository-url>
cd bobforge
```

2. **Setup Backend**
```bash
cd backend
npm install
cp .env.example .env
npm start
```
Backend runs on `http://localhost:3000`

3. **Setup Frontend** (in a new terminal)
```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```
Frontend runs on `http://localhost:5173`

4. **Open the app**
Navigate to `http://localhost:5173` in your browser

## 📖 Usage

### Generate a Blueprint

1. Open BobForge in your browser
2. Enter your project idea in the textarea
3. Click "Generate Blueprint" or try the example
4. Wait for the blueprint to be generated
5. View the complete blueprint on the dashboard

### Use with IBM Bob

1. Navigate to the "Bob Prompt" page
2. Copy the generated prompt
3. Open IBM Bob in your development environment
4. Paste the prompt and let Bob build your application
5. Iterate and refine with Bob's assistance

### Track Artifacts

1. Go to the "Artifacts" page
2. Add files generated by IBM Bob
3. Document purpose and status
4. Build evidence of Bob's contributions

## 🎨 User Flow

```
1. User enters project idea
   ↓
2. Frontend calls POST /api/blueprints/generate
   ↓
3. Backend generates complete blueprint
   ↓
4. Frontend stores blueprint ID in localStorage
   ↓
5. User views blueprint on dashboard
   ↓
6. User copies IBM Bob build prompt
   ↓
7. User pastes prompt into IBM Bob
   ↓
8. IBM Bob builds the application
   ↓
9. User tracks artifacts in BobForge
```

## 🔧 API Endpoints

### Blueprints
- `POST /api/blueprints/generate` - Generate new blueprint
- `GET /api/blueprints` - Get all blueprints
- `GET /api/blueprints/:id` - Get specific blueprint
- `GET /api/blueprints/:id/export` - Export as Markdown

### Artifacts
- `GET /api/artifacts` - Get all artifacts
- `POST /api/artifacts` - Create new artifact

### Health
- `GET /health` - Health check
